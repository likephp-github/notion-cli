Key design decisions implemented:
- Output contract: stdout JSON envelope on success, stderr JSON envelope on failure
- Exit codes: 0/1/2/3/4 for success/USER_ERROR/AUTH_ERROR/API_ERROR/NOT_FOUND
- Token storage: OS keyring primary, chmod-600 secrets.toml fallback (config.toml never has token)
- Tenacity retry: predicate-based (only 429/5xx/network), 3 attempts with exponential backoff
- Schema cache: ~/.cache/notion-cli/schema/<db_id>.json with 1h TTL, --no-cache override
- Type-aware coercion: 13 property types supported with options validation; --set-raw escape hatch
- Markdown converter: 8 block types round-trip; rich_text supports both 'plain_text' (response shape)
  and 'text.content' (request shape) for symmetric conversion
